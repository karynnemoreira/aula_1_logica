O que é lógica da programação?

Lógica da programação: Definir os passos e a ordem necessária para alcançarmos um objetivo. É a chave para aprender qualquer linguagem.

Um software segue uma série de operações em uma ordem definida para que ao fim o usuário alcance o seu objetivo. A lógica é antes de tudo um modo específico de organizar operações e chegar ao fim com um problema resolvido.

JavaScript

É uma Linguagem de programação interpretada estruturada, de script em alto nível com tipagem dinâmica fraca e multiparadigma. Juntamente com HTML e CSS, o JavaScript é uma das três principais tecnologias da World Wide Web.

  • Tipagem dinâmica fraca: Não é necessário informar o tipo das variáveis, além disso, a linguagem faz conversões entre os tipos automaticamente.
  • Multiparadigma: suportam vários paradigmas de programação.

O que é o console.log ??

É uma função nativa do JS que exibe uma mensagem no console do navegador. O acesso a esse console auxilia no processo de debugar, que é basicamente analisar o processo de Entrada > Processamento > Saída de partes do código. Será nosso melhor amigo durante a jornada com JS.

Por que colocar a tag script no final da body e não no head como o css??

Porque Colocar um script no head garante que ele seja executado antes de qualquer elemento ser colocado no body. Isso significa que ele garantidamente estará presente quando a página for "montada", ou seja, qualquer código que precise estar presente na hora de processar o body com certeza já estará pronto para agir. A desvantagem é que o usuário só vai ver uma página em branco até que o script termine de executar.

DECLARANDO VARIÁVEIS: VAR , LET , CONST

Antes de começarmos a falar sobre variáveis, vamos falar sobre o que é escopo, vamos ouvir isso algumas vezes na nossa vida de programador.

Escopo significa onde essas variáveis poderão ser utilizadas. Varias declarações têm escopo global ou função/escopo local.

O escopo é global é quando uma variável é declarada FORA DE UMA FUNÇÃO. Isso significa que qualquer variável declarada fora de um bloco de função está disponível para uso em todo o código.

Escopo local é quando uma variável é declarada DENTRO DE UMA FUNÇÃO, em bloco.

VARIÁVEIS

variável (VAR, LET OU CONST) + nome da variável (EX: ALUNO) = tipo de dados (numérico, "textual / string" ou boleando (verdadeiro ou falso) )

Ex¹: let aluno = "Gabriela" (textual, chamados de string)

EX²: let idade = 18 (numérico)

EX³ let amarelo = true (boleando)

São estruturas importantes em qualquer programação, independentemente da linguagem que se utilize para implementar os códigos ou programas. Elas podem ser entendidas como uma caixinha, onde os dados são armazenados temporariamente ou em definitivo e que são manipuladas durante a execução do programa.

  • VAR: Antes do advento do ES6, as declarações de var eram a regra. Havia problemas associados à declaração de variáveis com var, no entanto. Por isso, foi necessário o surgimento de novas formas de declarar variáveis. *Podem ser atualizadas e declaradas novamente dentro de seu escopo.
  • LET: Declara uma variável local, mais usada, é a preferida entre os programadores para declaração de variáveis. Uma variável declarada em um bloco com let só está disponível para uso dentro desse bloco. *Podem ser atualizadas, mas não podem ser declaradas novamente.
  • CONST: Não pode ser atualizado nem declarado novamenteman, têm valores constantes. Declarações com const compartilham algumas semelhanças com as declarações com let.Declarações com const têm escopo de bloco. *Não podem ser atualizadas nem declaradas novamente.
